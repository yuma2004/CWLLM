# 懸念点

- 本番運用では `JWT_SECRET` / DB認証情報 / LLMキーの安全な管理が必須（`.env` 直置きは非推奨）。
- `docker-compose.prod.yml` のDBはローカル向け設定のため、実運用では外部/マネージドDBに置き換えが必要。
- CSVエクスポートは全件取得のため、データ量が増えると性能・メモリ負荷が高くなる可能性。
- 監査ログは更新系のみで、参照系はアプリログにのみ残る（永続化要件があるなら別途設計が必要）。
- LLM APIは外部通信前提のため、APIキー未設定時はモック動作になる（本番はキー投入必須）。
- バックアップ/リストアは `pg_dump` / `pg_restore` 依存で、実行環境にPostgreSQLクライアントが必要。
- Docker構成ではマイグレーション/seedを自動実行しないため、初回デプロイ時の手順が必要。
