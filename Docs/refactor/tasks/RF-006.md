# RF-006: useApi 整理

Title: useApi hooks を整理して責務を分ける
Lane: SOLO
Scope: frontend/src/hooks/useApi.ts, frontend/src/hooks/useApiClient.ts
Forbidden:
- 既存APIのリクエスト/レスポンス変更
- frontend/src/pages や backend/ の変更
Diff budget: 2 files, <=200 lines moved/edited

Preconditions:
- 事前検証(RUNBOOK Step1) が PASS

Steps:
1. `frontend/src/hooks/useApiClient.ts` を整理
2. useApi.ts の責務を整理して分割
3. useApi.ts の型/エラー処理を整理
4. 既存 import の置き換えを実施
5. typecheck/test を実行

Acceptance Criteria:
- useFetch/useMutation の挙動は不変
- 既存の呼び出しが壊れていない
- typecheck/test が PASS

Verification:
- npm --prefix frontend run typecheck
- npm --prefix frontend run test -- --run

Risk: Medium - hooks 周辺
Rollback:
- git revert <commit>
